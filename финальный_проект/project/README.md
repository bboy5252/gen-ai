# MarketTwits Event Radar

Финальный проект по треку B курса «Практическое применение генеративного ИИ».

Проект анализирует публичные посты канала `https://t.me/s/markettwits` и для каждой новости определяет:

- тему новости;
- связанную компанию, тикер или рыночный инструмент;
- примерное влияние: позитивное, негативное, нейтральное, смешанное или без прямой привязки к компании;
- короткое объяснение;
- дословную цитату из исходного поста;
- трассу инструментов, которые были использованы при обработке.

## Установка

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Одна команда запуска

```bash
python3 run_all.py --offline --limit 40
```

После запуска артефакты появятся в папке `output/`.

## Режим с LLM

По умолчанию проект можно запускать без ключей в детерминированном offline-режиме. Если нужно включить LLM-вызовы, скопируйте `.env.example` в `.env`, поставьте `OFFLINE=0` и заполните один из вариантов:

- `OPENAI_API_KEY`;
- или `LLM_BASE_URL`, `LLM_AUTH_TOKEN`, `LLM_MODEL`.

Затем запустите:

```bash
python3 run_all.py --limit 40
```

## Что Где Лежит

- `fetch_markettwits.py` — загрузчик публичной web-версии Telegram-канала.
- `schema.py` — Pydantic-схемы структурированного результата и бизнес-валидаторы.
- `rag.py` — BM25-поиск по правилам рынка и каталогу компаний.
- `market_tools.py` — инструменты для поиска тикеров, тем, инструментов и направления влияния.
- `agent.py` — обработка одного поста: RAG, tools, structured output, judge.
- `pipeline.py` — полный прогон по корпусу постов.
- `eval.py` — проверочный набор размеченных примеров.
- `run_all.py` — единая команда для запуска пайплайна и проверки.
- `input/posts_sample.json` — сохраненный срез постов MarketTwits.
- `input/company_catalog.json` — мини-каталог компаний, тикеров и рыночных инструментов.
- `input/market_rules.md` — правила классификации тем и влияния.
- `output/analysis.json` и `output/analysis.csv` — результаты анализа постов.
- `output/summary.json` — агрегированная сводка по прогону.
- `output/trace.jsonl` — пошаговая трасса работы инструментов.
- `output/hallucination_report.json` — проверка цитат и чисел на выдумывание.
- `отчёт.md` — описание того, что сделано и как устроен проект.

## Коротко О Логике

Сначала пост проходит через RAG по правилам и каталогу компаний. Затем агент вызывает инструменты: ищет тикеры, определяет тему, выделяет инструменты, направление влияния и срочность. После этого результат собирается в строгую Pydantic-схему, а отдельный judge проверяет, что цитаты и числа действительно есть в исходном тексте.
