## 3. Диагностика ошибок

### Ошибка 1. Битый формат tool-call

Трасса:

```json
{"run_id":"6d049b73-4c1f-406f-a890-3ca22b537265","step":1,"final":"BadRequestError ... tool_use_failed ... <function=get_fx_rate>{\"currency\":\"USD\",\"on_date\":\"2026-06-16\"}<function=get_fx_rate>{\"currency\":\"USD\",\"on_date\":\"2022-01-01\"}","error":"BadRequestError ..."}
{"run_id":"9860b734-a070-473b-a207-09653624914e","step":1,"final":"BadRequestError ... tool_use_failed ... <function=calculate>{\"72 / (get_key_rate{on_date: '2026-06-16'} / 100)\"}</function>","error":"BadRequestError ..."}
```

Причина: модель сгенерировала псевдо-XML `<function=...>` вместо штатного tool-call JSON API, поэтому запрос упал до исполнения инструментов.

Что исправить: при `tool_use_failed` делать автоматический повтор с коротким системным напоминанием про формат tool-call; дополнительно можно выбрать модель, которая стабильнее поддерживает function calling.

### Ошибка 2. Неправильный порядок вызовов

Трасса:

```json
{"step":1,"call":"get_key_rate","args":{"on_date":"2026-06-16"},"obs":{"rate":16.0,"source":"fallback_csv"}}
{"step":1,"call":"get_inflation","args":{"month":6,"year":2026},"obs":{"error":"нет данных ИПЦ на 2026-06"}}
{"step":1,"call":"calculate","args":{"expression":"get_key_rate(on_date='2026-06-16') - get_inflation(month=6, year=2026)"},"obs":{"error":"ValueError: запрещённый фрагмент выражения: Call(... get_key_rate ...)"}}
```

Причина: агент одновременно вызвал источники данных и калькулятор, не дождавшись чисел. В `calculate` попали названия инструментов, а не готовое числовое выражение.

Что исправить: разделить шаги жестче: сначала данные, затем отдельный шаг арифметики. Можно добавить валидатор, который при наличии `get_` в выражении возвращает более явную подсказку агенту.

### Ошибка 3. Недоступный месяц в данных

Трасса:

```json
{"step":1,"call":"get_inflation","args":{"month":6,"year":2026},"obs":{"error":"нет данных ИПЦ на 2026-06"}}
{"step":2,"call":"get_inflation","args":{"month":5,"year":2026},"obs":{"error":"нет данных ИПЦ на 2026-05"}}
{"step":3,"call":"get_inflation","args":{"month":3,"year":2026},"obs":{"year":2026,"month":3,"cpi_yoy":6.8,"source":"rosstat_csv"}}
```

Причина: для инфляции в CSV последний доступный месяц - 2026-03, но агент сначала пытался взять текущий и предыдущий месяц.

Что исправить: сделать отдельный инструмент `get_latest_available_period(metric)` или возвращать из `get_inflation` ближайший доступный месяц с явной пометкой.

### Ошибка 4. Бесконечный цикл до max_iter

Трасса:

```json
{"step":1,"call":"calculate","args":{"expression":"100 * (1 + (get_key_rate/on_date=null)/100) / (1 + get_inflation/month=3/year=2026/100) - 1"},"obs":{"error":"SyntaxError: invalid syntax (<unknown>, line 1)"}}
{"step":4,"call":"calculate","args":{"expression":"100 * (1 + (get_key_rate/on_date=null)/100) / (1 + get_inflation/month=3/year=2026/100) - 1"},"obs":{"error":"SyntaxError: invalid syntax (<unknown>, line 1)"}}
{"step":5,"final":"исчерпан лимит шагов max_iter=5","error":"исчерпан лимит шагов max_iter=5"}
```

Причина: агент повторял одно и то же невалидное выражение, не используя уже полученные значения `16.0` и `6.8`.

Что исправить: хранить последнюю ошибку и запрещать повторять тот же вызов без изменения аргументов; после двух одинаковых ошибок возвращать диагностический финал.
