## 1. Валидатор схемы

Сигнатура:

```python
def validate_plan(plan: Plan) -> list[str]:
```

Разрешенные инструменты:

```python
VALID_TOOLS = {"get_fx_rate", "get_key_rate", "get_inflation", "calculate"}
```

Пойманные примеры:

```text
подвопрос 1: неизвестный инструмент get_cumulative_inflation
подвопрос 2: expected_tools должен содержать только имя, не вызов: get_fx_rate(currency="USD")
```

В `run_pwc()` валидатор вызывается сразу после `planner(...)`. Если есть ошибки,
планировщик вызывается повторно с feedback. Если после feedback план все еще
невалидный или пустой, PWC возвращает честный отказ вместо передачи
галлюцинированного инструмента исполнителю.

## 3. Замер критики

Добавлен `critic_experiment.py` с 5 заведомо битыми кейсами:

- арифметика без `calculate`
- выдуманное число
- несогласованные данные
- ошибка исполнителя
- план не покрывает вопрос

Полная команда:

```bash
python critic_experiment.py -n 10
```

Попытка полного прогона через OpenRouter `nex-agi/nex-n2-pro:free` показала
важную совместимость: часть free-провайдеров не принимает запросы, где всё
содержимое лежит только в `system`. После этого `critic.py` исправлен: основной
текст проверки теперь отправляется как `user`-сообщение.

Файл `critic_experiment_results.json` фиксирует не ложные принятия, а ошибки
провайдера: по 20 ошибок на каждый битый кейс. После исправления повторить
полный N=10 на этом ключе не удалось, потому что OpenRouter free исчерпал
дневной лимит `free-models-per-day`.

Последний валидный короткий smoke до полного OpenRouter-прогона был:

| Битый кейс | T=0.0, ложных принятий | T=0.7, ложных принятий |
|---|---:|---:|
| арифметика без calculate | 0/1 | 0/1 |
| выдуманное число | 0/1 | 0/1 |
| несогласованные данные | 0/1 | 0/1 |
| ошибка исполнителя | 0/1 | 0/1 |
| план не покрывает вопрос | 0/1 | 0/1 |

Гипотеза «шум лечит зеркальное соглашение» требует повторного полного запуска
после сброса лимита или пополнения OpenRouter credits.

Все PWC-падения в этом smoke-прогоне связаны с `RateLimitError`, а не с

моделей: `free-models-per-day`. Полный требуемый запуск:

```bash
export LLM_BASE_URL=https://openrouter.ai/api/v1
export LLM_MODEL=nex-agi/nex-n2-pro:free
export LLM_CALL_INTERVAL_SEC=3.5
python eval_pwc.py -n 5
```

## 5. Вывод

PWC оправдан, когда вопрос естественно раскладывается на независимые факты и
явную арифметику: курсы на разные даты, ставка плюс инфляция, несколько
валютных запросов. Он хуже одиночного агента, когда планировщик начинает
выдумывать tools или когда внешний LLM нестабилен: тогда нужен валидатор,
лимиты на retry и честный отказ вместо попытки любой ценой получить ответ.
